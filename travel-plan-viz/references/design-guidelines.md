# 内置美学准则（无 frontend-design 时的兜底）

设计步骤优先调用专业设计 skill（`frontend-design` 或 `huashu-design` 花叔Design，任一已安装即用）；**两者都未安装**时，按本准则自己出风格化 HTML，目标是"有性格、不像 AI 默认模板"。这是本仓库自有的精简指导，使 skill 可独立使用。

## 配色

- 每趟行程选一套**有性格的配色**（深色背景 + 主题强调色，或清爽浅色 + 一个鲜明主色），不同行程用不同配色以便区分。
- 用 CSS 变量统一定义（如 `--bg`、`--bg-card`、`--text`、`--text-dim`、`--accent`、`--border`），全局复用。
- 保证文字与背景**高对比**，可读性第一。

## 排版与布局

- **响应式，移动 + 桌面都要好看**：手机端单列、主体约 420–480px；**桌面端（≥768px）容器加宽到约 880–960px 居中**，卡片列表（航班/酒店/行前须知/餐饮/景点）用多列网格、地图更宽。务必写 `@media` 断点，别让大屏只剩一条窄列。
- 层级分明：标题用有个性的字体（衬线/展示体皆可），正文清爽易读；字号、字重拉开层级。
- **卡片化**：分区用卡片承载，圆角、充足留白、细边框或极淡阴影，不要拥挤。
- 一致的视觉语言：图标、徽章、标签"药丸"风格统一（如营业时间/门票/交通用同一套小标签）。

## 细节

- 图片**严禁变形**：容器固定尺寸/比例，`<img>` 用 `object-fit: cover` + `display:block`（推荐绝对定位 `inset:0` 填满）。
- 关键信息（待办清单、⚠️ 提醒徽章、已预订标识、免责声明）用颜色/图标突出，但克制，不要花哨到喧宾夺主。
- 适度的过渡/悬停反馈即可，避免炫技动画拖慢手机。
- 整体追求"精心设计过"的观感，避免千篇一律的默认灰白卡片。

> 装了 frontend-design 或 huashu-design 会更出彩；本准则只是保证"没装也能拿到一份像样、好看的页面"。
